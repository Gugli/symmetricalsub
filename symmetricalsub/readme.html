Symmetrical Subscription Plugin Readme

Overview

When installed, this plugin only allow bidirectional subscriptions. Basically, what it does is : 1 - When an user accepts a sub request, he automatically sends one back. 2 - When an user is subscribed (or requesting subscription) to a contact, he automatically accept subscription requests from that contact 3 - When an user recieve an unsubscribe request, he automatically sends one (if he's subscribed) or cancel any pending subscription requests Since it does not make sense when interaction with other domains (which may not have the same policy), it can be enabled for local requests only.

Installation

Copy the file, symmetricalsub.jar into the plugins directory of your Openfire installation. The plugin will then be automatically deployed. To upgrade to a new version: 1) go to the plugin screen of the Admin Console, 2) click on the delete icon on the same row as the currently installed symmetricalsub plugin, 3) copy the new symmetricalsub.jar into the plugins directory of your Openfire installation.

Configuration

The symmetricalsub plugin is configured via Symmetrical Subscription Properties sidebar item which can found under the Sever tab in Openfire Admin Console. The symmetricalsub plugin can be configured in the following ways:

  • Disabled (Default setting) - The server allows asymmetrical relationship.

  • Enabled - All relationships are symmetrical.
    • Local - Only subscription requests sent by users who have an account on the domain on which the plugin is installed, will be symmetrical.
    • All - All subscription requests, even to other domains, are treated as if symmetrical.

Using the Plugin

Once the plugin has been installed and configured there is no additional steps are required to use the plugin.
